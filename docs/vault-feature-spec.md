# FlaskDrop 保险箱功能需求说明书

**版本：** v1.0  
**日期：** 2026-04-13  
**状态：** 待评审

---

## 1. 背景与目标

FlaskDrop 现有功能基于 WebRTC P2P，要求双方同时在线。**保险箱（Vault）** 功能补充了异步传递场景：存入方将内容存到服务端，生成一次性收件码，取件方凭码在任意时间、任意平台取走内容，取走后内容自动销毁。

**目标：**
- 支持文本和文件的异步单次传递
- 内容一次性，取走即销毁，保护隐私
- 收件码短小易传，跨平台可用（短信、微信、邮件均可）

---

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| **存入方** | 将内容存入保险箱，获得收件码 |
| **取件方** | 凭收件码查看或下载内容，确认后内容销毁 |
| 两者可以是同一人（自己存、自己在另一设备取） |

---

## 3. 功能需求

### 3.1 入口

- 主界面（房间屏）右上角或固定位置显示「🔒 保险箱」按钮
- 点击后切换到保险箱存入界面（不离开当前房间，覆盖层或独立页均可）
- 保险箱界面也可在未加入房间时独立访问（路由：`/vault`）

### 3.2 存入流程

**步骤：**

1. 用户选择内容类型：**文本** 或 **文件**
2. 输入内容：
   - 文本：多行文本框，无字数上限（建议软限制 50,000 字符）
   - 文件：拖拽或点击选择，单文件，大小 ≤ 100 MB
3. 有效期固定 **30 分钟**（倒计时显示）
4. 点击「存入保险箱」
5. 上传进度条（文件场景）
6. 成功后展示**收件码**（格式见 3.5）
7. 提供「复制收件码」按钮，一键复制

**存入界面字段：**

```
┌─────────────────────────────────────┐
│  🔒 保险箱                    [×]   │
├─────────────────────────────────────┤
│  [文本]  [文件]                      │
│                                     │
│  ┌─────────────────────────────┐    │
│  │ 在此输入文本内容...          │    │
│  └─────────────────────────────┘    │
│                                     │
│  有效期：[24小时 ▼]                  │
│                                     │
│  [  存入保险箱  ]                    │
└─────────────────────────────────────┘
```

### 3.3 收件码展示

存入成功后：

```
┌─────────────────────────────────────┐
│  ✅ 已存入保险箱                     │
│                                     │
│         收件码                       │
│      ┌──────────┐                   │
│      │  A3K9F2  │  [复制]           │
│      └──────────┘                   │
│                                     │
│  有效期至：2026-04-14 22:04          │
│  取件后自动销毁                       │
│                                     │
│  [再存一个]    [关闭]                │
└─────────────────────────────────────┘
```

### 3.4 取件流程

**入口：**
- 主界面「🔒 保险箱」→ 切换到「取件」Tab
- 独立路由：`/vault/pickup`
- 直接访问：`/vault/<code>`（URL 携带收件码，自动填入）

**步骤：**

1. 输入 6 位收件码
2. 点击「查看内容」
3. 服务端验证：
   - 码存在且未过期 → 返回内容
   - 码不存在或已过期 → 提示错误
4. 展示内容：
   - 文本：直接显示，提供「复制」按钮
   - 文件：显示文件名、大小、类型，提供「下载」按钮
5. 用户点击「✅ 确认收取，销毁内容」
6. 服务端删除内容，展示「已销毁」提示

**取件界面：**

```
┌─────────────────────────────────────┐
│  🔒 保险箱取件                       │
│                                     │
│  收件码：[        ]  [查看]          │
│                                     │
│  ─────────────────────────────────  │
│  📄 文件名：report.pdf               │
│  大小：2.3 MB                        │
│  存入时间：2026-04-13 22:04          │
│  有效期至：2026-04-14 22:04          │
│                                     │
│  [  ⬇ 下载文件  ]                   │
│                                     │
│  [✅ 确认收取，销毁内容]              │
└─────────────────────────────────────┘
```

### 3.5 收件码规格

| 属性 | 规格 |
|------|------|
| 长度 | 6 位 |
| 字符集 | 纯数字 `0123456789` |
| 空间 | 10^6 = 100 万 |
| 格式 | 纯随机，不含语义 |
| 有效期 | 30 分钟（1800 秒） |

### 3.6 内容生命周期

```
存入 → [有效期内，等待取件] → 取件确认 → 立即销毁
                ↓
         超过有效期（定时任务清理）→ 自动销毁
```

| 事件 | 行为 |
|------|------|
| 取件方点击「确认收取」 | 立即删除内容和收件码记录 |
| 超过有效期 | 定时任务（每小时）清理过期内容 |
| 存入方主动撤销（可选） | 提供撤销入口，需要原收件码确认 |

## 4. 非功能需求

### 4.1 安全

- 收件码不可被暴力枚举：连续错误 5 次后，同 IP 锁定 10 分钟
- 文件存储路径不可被直接猜测（使用 UUID 命名，不暴露原始文件名）
- 文本内容存储时可选加密（AES-256，密钥由收件码派生）
- HTTPS 环境下传输

### 4.2 性能

- 文件上传支持进度显示
- 单文件上限 100 MB，超出时前端拦截并提示
- 服务端存储目录可配置（默认 `./vault_storage/`）

### 4.3 存储

- 文件：存储在服务端本地目录，文件名为 UUID
- 文本：存储在 SQLite `vault_items` 表
- 元数据：收件码、类型、文件路径/文本内容、过期时间、状态

### 4.4 兼容性

- 移动端友好（取件码输入框大字体，方便手动输入）
- 支持 URL 直达：`/vault/A3K9F2` 自动填入收件码

## 5. 数据模型

### vault_items 表

```sql
CREATE TABLE vault_items (
    id          TEXT PRIMARY KEY,        -- UUID
    code        TEXT UNIQUE NOT NULL,    -- 6位收件码（大写）
    type        TEXT NOT NULL,           -- 'text' | 'file'
    content     TEXT,                    -- 文本内容（type=text时）
    file_path   TEXT,                    -- 文件存储路径（type=file时）
    file_name   TEXT,                    -- 原始文件名
    file_size   INTEGER,                 -- 文件大小（字节）
    mime_type   TEXT,                    -- MIME类型
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at  TIMESTAMP NOT NULL,      -- 过期时间
    status      TEXT DEFAULT 'active'    -- 'active' | 'claimed' | 'expired'
);
```

## 6. API 设计

| 方法 | 路径 | 描述 |
|------|------|------|
| `POST` | `/api/vault` | 存入内容（multipart/form-data） |
| `GET` | `/api/vault/<code>` | 查询收件码（返回元数据，不含内容） |
| `GET` | `/api/vault/<code>/content` | 获取文本内容 |
| `GET` | `/api/vault/<code>/download` | 下载文件 |
| `DELETE` | `/api/vault/<code>` | 确认收取，销毁内容 |

### POST /api/vault 请求

```
Content-Type: multipart/form-data

type=text&content=Hello World&expires_in=86400
或
type=file&file=<binary>&expires_in=86400
```

### POST /api/vault 响应

```json
{
  "ok": true,
  "data": {
    "code": "A3K9F2",
    "expires_at": "2026-04-14T22:04:00Z",
    "type": "text"
  }
}
```

## 7. UI 交互细节

### 7.1 保险箱按钮位置

- 房间屏右上角，与「离开房间」按钮并排
- 图标：🔒 或锁形 SVG
- 未加入房间时，在加入/创建界面也显示入口

### 7.2 文件上传体验

- 拖拽区域与现有 peer card 的传输区风格一致
- 上传中显示进度条（百分比 + 速度）
- 超过 100 MB 时前端立即拦截，不发起请求

### 7.3 收件码输入体验

- 输入框自动转大写
- 6 位自动提交（无需点击按钮）
- 支持粘贴后自动去除空格和连字符

### 7.4 销毁确认

- 「确认收取」按钮需二次确认（弹出提示：「内容将立即销毁，无法恢复」）
- 销毁后展示动画（锁关闭效果）

## 9. 开发优先级

### P0（核心功能，必须实现）
- 文本存入 + 收件码生成
- 文件存入（≤100MB）+ 收件码生成
- 凭码查看/下载
- 确认收取后销毁
- 过期自动清理

### P1（体验优化）
- 上传进度条
- URL 直达（`/vault/<code>`）
- 收件码 6 位自动提交
- 移动端适配

### P2（安全增强）
- 错误次数限制（防暴力枚举）
- 文本加密存储
- 存入方主动撤销

## 10. 技术实现要点

基于现有 FlaskDrop 技术栈（Python + Flask + SQLite + 原生 JS）：

- **后端**：新增 `vault.py` 模块，注册到 `app.py`；文件存储在 `vault_storage/` 目录
- **前端**：新增 `vault.js`；保险箱界面作为覆盖层（overlay）叠加在现有界面上
- **定时清理**：复用现有 `threading.Timer` 机制，每小时扫描过期 vault_items
- **文件上传**：使用 `fetch` + `FormData`，`XMLHttpRequest` 监听 `progress` 事件显示进度
